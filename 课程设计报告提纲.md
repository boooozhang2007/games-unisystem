## 一、需求分析

### 1.系统概述

随着网络游戏的发展，越来越多的玩家会在游戏中进行娱乐、交友和交易。不同年龄、不同身份、不同兴趣的用户，适合的游戏类型也不一样。比如有些用户喜欢竞技类游戏，有些用户更喜欢冒险、养成或者休闲类游戏。另外，很多游戏还带有道具、积分、皮肤、卡牌等资产内容，玩家之间会产生一定的交易需求。因此，本次课程设计选择开发一个“网络游戏推荐与交易系统”，通过用户的年龄、身份或职业、爱好、游戏时长、胜率等信息来进行推荐，并支持好友管理、交易管理和排行榜等功能。

本系统采用 C 语言实现，运行在控制台环境下，所有数据都保存到本地二进制文件中。系统主要面向两类角色：普通玩家和管理员。普通玩家可以注册、登录、查看和修改自己的信息、查看推荐游戏、管理好友、发起好友邀约、进入市场交易道具等；管理员在拥有普通功能的基础上，还能够查看全部用户、升级用户、降级用户和删除用户。

本系统在设计时尽量按照题目要求进行实现，加入了验证码登录、用户画像字段、自动升级、交易记录、好友邀约等内容，使系统更贴近“网络游戏推荐系统（可扩展为游戏装备交易等）”这一题目要求。

### 2.系统运行环境

#### （1）硬件环境

- 普通 PC 机即可运行
- CPU：Intel i5 或同等级别处理器
- 内存：4GB 及以上
- 硬盘：能够保存程序文件和数据文件即可

#### （2）软件环境

- 操作系统：Windows 10 / Windows 11
- 编译器：GCC
- 开发工具：Code::Blocks、VS Code 等
- 程序形式：C 语言控制台程序

### 3.功能需求描述

根据题目要求，本系统主要需要实现以下几个方面的功能：

#### （1）用户注册功能

用户可以输入用户名、密码、年龄、身份或职业、爱好等信息进行注册。系统需要检查用户名是否重复，如果用户名已经存在，就不能重复注册；如果用户名没有被占用，就可以把新用户信息保存到文件中。

#### （2）用户登录功能

用户通过输入用户名和密码进行登录。为了更贴近题目要求，本系统增加了验证码功能。用户在密码输入正确之后，还需要继续输入验证码，验证码正确后才能真正登录成功。

#### （3）用户信息管理功能

用户登录后可以查看自己的完整信息，并能够修改密码、年龄、身份或职业、爱好、游戏时长、对局数、胜场数、金币等内容。管理员可以查看全部用户信息，并对用户进行升级、降级和删除操作。

#### （4）用户画像功能

系统记录用户的年龄、身份或职业、爱好、游戏时长、总对局、胜场、胜率等信息。这些信息不仅用于显示，还会参与推荐和自动升级逻辑。

#### （5）推荐功能

系统能够根据用户的年龄、身份或职业、爱好、游戏时长等信息推荐不同类型的游戏。例如：

- 竞技爱好用户更容易被推荐竞技类游戏；
- 游戏时长较高的用户更可能被推荐竞技或深度内容游戏；
- 不同身份或职业的用户也会有一定推荐倾向。

为了方便展示，程序内置了一些国内热门网络游戏数据，如王者荣耀、原神、第五人格、和平精英、蛋仔派对、阴阳师等。

#### （6）自动升级功能

系统支持根据游戏时长和胜率进行自动升级。如果普通用户游戏时长达到一定标准，或者胜率达到一定标准，系统会自动把该用户升级为 VIP 用户。

#### （7）好友管理功能

用户可以添加好友、删除好友、查看好友列表。同时，系统还支持好友邀约功能，用户可以选择自己的好友并发起某个游戏的邀约，邀约记录会被保存到文件中，方便后续查看。

#### （8）交易功能

系统支持简单的游戏道具交易，包括：

- 查看市场中的道具；
- 发布道具到市场；
- 购买其他用户上架的道具；
- 查看与自己有关的交易记录。

每次购买成功后，系统会自动更新买卖双方金币，并保存一条交易记录。

#### （9）其他功能

系统还实现了金币排行榜，能够将用户按照金币数量从高到低排序并显示。这样可以让系统更有完整性，也更符合题目中“其他相关功能”的要求。

### 4.数据存储需求

本系统使用二进制文件保存数据，主要数据文件如下：

- `users.dat`：保存用户信息
- `games.dat`：保存游戏信息
- `market.dat`：保存市场道具信息
- `trade_log.dat`：保存交易记录
- `invite_log.dat`：保存好友邀约记录
- `id.dat`：保存当前编号
- `seed.dat`：保存验证码伪随机种子

通过这些文件，系统可以在关闭程序后保留数据，下次启动程序时继续使用之前保存的信息。

## 二、总体设计

### 1.总体开发思想

本系统采用模块化设计思想，把程序分成四个主要部分：crud 模块、manage 模块、trade 模块和 main 模块。这样做的好处是结构比较清楚，写代码时不容易混乱，后面查找和修改问题也比较方便。

- **crud 模块**：主要负责文件读写和基础数据操作，例如读取用户、保存用户、读取市场、保存市场、生成新编号等。
- **manage 模块**：主要负责用户相关功能，例如注册、登录、个人信息管理、自动升级、推荐功能、好友功能等。
- **trade 模块**：主要负责市场相关功能，例如查看市场、上架道具、购买道具、查看交易记录等。
- **main 模块**：主要负责主菜单、排行榜和程序入口控制。

程序整体运行流程是：

1. 程序启动；
2. 初始化数据文件；
3. 如果没有用户则自动创建管理员账号；
4. 用户输入用户名；
5. 如果用户存在则登录，否则进入注册；
6. 登录成功后进入主菜单；
7. 根据菜单选择进入不同功能模块；
8. 退出程序。

### 3.模块设计说明

#### （1）crud 模块

这一部分主要是程序最底层的数据操作。因为本系统的数据都是存在文件里的，所以每次做用户登录、市场显示、交易保存等操作时，几乎都会先调用 crud 模块的函数。该模块的主要任务包括：

- 初始化数据文件；
- 生成新的编号；
- 读取和保存用户数据；
- 读取和保存游戏数据；
- 读取和保存市场数据；
- 保存交易记录和邀约记录；
- 根据用户名、用户编号、道具编号进行查找。

#### （2）manage 模块

这一部分主要处理“人”的事情，也就是用户和好友的逻辑。该模块实现了：

- 注册用户；
- 登录用户；
- 验证码检查；
- 显示个人信息；
- 修改个人信息；
- 自动升级；
- 推荐游戏；
- 添加好友；
- 删除好友；
- 显示好友；
- 发起好友邀约；
- 查看邀约记录；
- 管理员管理用户。

#### （3）trade 模块

这一部分主要负责市场和交易。用户可以把自己的道具放到市场里，也可以从市场里购买其他用户的道具。购买完成之后，系统会自动记录交易信息。该模块让系统更贴近“游戏装备交易”的扩展要求。

#### （4）main 模块

main 模块负责整个程序的运行流程控制。包括：

- 程序启动时初始化；
- 是否进入登录还是注册；
- 显示主菜单；
- 跳转到排行榜、推荐、好友、交易等功能；
- 最终退出程序。

## 三、数据结构设计

### 1.用户结构体设计

系统中最重要的数据结构之一就是用户结构体：

```c
typedef struct {
    int id;
    char username[50];
    char passwd[50];
    int age;
    char identity[30];
    char hobby[30];
    int role;
    int coins;
    int game_hours;
    int play_count;
    int win_count;
    int friend_count;
    int asset_count;
    int friends[MAX_FRIENDS];
    int assets[MAX_ASSETS];
} users;
```

这个结构体里既保存了账号信息，也保存了用户画像信息，还保存了好友和资产关系，因此是整个系统中最核心的数据结构。

字段说明如下：

- `id`：用户编号
- `username`：用户名
- `passwd`：密码
- `age`：年龄
- `identity`：身份或职业
- `hobby`：爱好
- `role`：角色（普通、VIP、管理员）
- `coins`：金币
- `game_hours`：游戏时长
- `play_count`：总对局数
- `win_count`：胜场数
- `friend_count`：好友数量
- `asset_count`：拥有资产数量
- `friends[]`：好友编号数组
- `assets[]`：资产编号数组

### 2.游戏结构体设计

```c
typedef struct {
    int game_id;
    char game_name[50];
    char game_type[30];
    char game_desc[100];
    int need_age;
} games;
```

这个结构体用来保存游戏的基础信息，主要是为了推荐系统服务。

字段说明：

- `game_id`：游戏编号
- `game_name`：游戏名称
- `game_type`：游戏类型
- `game_desc`：游戏描述
- `need_age`：建议年龄

### 3.市场结构体设计

```c
typedef struct {
    int item_id;
    char item_name[50];
    char item_type[30];
    int item_price;
    int seller_id;
} market;
```

该结构体用于保存市场中正在出售的道具信息。购买时会根据这个结构体中的信息判断价格和卖家。

### 4.交易记录结构体设计

```c
typedef struct {
    int buyer_id;
    int seller_id;
    char buyer_name[50];
    char seller_name[50];
    int item_id;
    char item_name[50];
    char item_type[30];
    int item_price;
} trade_log;
```

这个结构体用来保存每一次交易的结果，便于用户之后查看自己的交易历史。

### 5.好友邀约记录结构体设计

```c
typedef struct {
    int send_id;
    int recv_id;
    char send_name[50];
    char recv_name[50];
    char game_name[50];
} invite_log;
```

这个结构体用来保存好友邀约信息，让系统在“好友管理”这部分更贴近真实游戏场景。

### 6.数据组织方式说明

本系统没有使用链表，而是采用了“结构体数组 + 二进制文件”的方式：

- 文件中保存长期数据；
- 程序运行时把数据读入数组；
- 修改后再整体写回文件。

这样做的原因是：

1. 更符合当前课程学习阶段；
2. 比较容易理解和调试；
3. 也能够满足课程设计对文件读写和数据结构的要求。

## 五、功能实现

### 1.主要函数说明

#### （1）crud 模块主要函数

**1）void crud_init(void)**

功能：初始化系统运行时需要使用的二进制数据文件，如果文件不存在则创建。该函数还负责准备编号文件和验证码种子文件，并调用默认游戏写入函数。

**2）void crud_write_default_games(void)**

功能：向游戏文件中写入默认热门游戏数据。如果发现当前游戏文件已经是新版默认数据，则不重复写入。

**3）int crud_next_id(void)**

功能：读取编号文件中的当前编号，并生成一个新的编号返回，同时把最新编号重新写回文件。

**4）int crud_load_users(users list[])**

功能：从 `users.dat` 中读取全部用户数据，保存到数组中，并返回读到的用户数量。

**5）void crud_save_users(users list[], int num)**

功能：把用户数组整体写回 `users.dat` 文件，保证最新用户数据被保存下来。

**6）int crud_load_games(games list[])**

功能：从 `games.dat` 中读取全部游戏数据。

**7）int crud_load_market(market list[])**

功能：从 `market.dat` 中读取市场中的全部道具。

**8）void crud_save_market(market list[], int num)**

功能：把市场中的道具数组写回 `market.dat`。

**9）void crud_add_trade_log(trade_log* log_data)**

功能：追加写入一条交易记录。

**10）void crud_add_invite_log(invite_log* log_data)**

功能：追加写入一条好友邀约记录。

#### （2）manage 模块主要函数

**1）int manage_user_register(users* user)**

功能：注册用户，检查用户名是否重复，设置初始角色和金币，然后把用户信息保存到文件中。

**2）int manage_user_login(char username[], char passwd[])**

功能：登录用户。先判断用户名是否存在，再判断密码是否正确，最后要求输入验证码，三者都正确后才算登录成功。

**3）int manage_make_captcha(char username[], int id)**

功能：生成一个四位数的伪随机验证码，并把新的种子值写回文件，使验证码每次登录都会变化。

**4）void manage_user_show_info(char username[])**

功能：显示用户的详细信息，包括画像内容、游戏时长、对局数、胜率、好友数和资产数等。

**5）int manage_user_edit_info(char username[])**

功能：修改个人信息，包括密码、年龄、身份或职业、爱好、游戏时长、胜负记录和金币。

**6）void manage_user_auto_upgrade(users* user)**

功能：根据用户的游戏时长和胜率判断是否自动升级为 VIP。

**7）int manage_friend_add(char username[])**

功能：添加好友，双方会互相进入对方的好友列表。

**8）int manage_friend_remove(char username[])**

功能：删除好友，也是双向删除。

**9）int manage_friend_invite(char username[])**

功能：向好友发起游戏邀约，并保存邀约记录。

**10）void manage_show_recommend_games(char username[])**

功能：根据用户画像和游戏信息进行简单推荐。

#### （3）trade 模块主要函数

**1）void trade_show_market(void)**

功能：显示当前市场中所有可购买的道具信息。

#### （4）main 模块主要函数

**1）int main_show_menu(int role)**

功能：显示主菜单，不同角色显示的菜单内容略有区别。

**2）void main_show_ranking(void)**

功能：显示金币排行榜，内部采用最基础的冒泡排序实现。

**3）void main_first_register(char username[])**

功能：当输入的用户名不存在时，进入首次注册流程。
