## 一、需求分析

### 1.系统概述

随着网络游戏产业的不断发展，网络游戏已经成为当代用户常见的娱乐方式之一。当前市面上的网络游戏种类丰富，既有适合青少年玩家的休闲类游戏，也有适合成年用户的竞技类、冒险类和养成类游戏。与此同时，玩家在游戏过程中不仅会产生娱乐需求，还会形成社交需求与交易需求，例如添加好友、发起邀约、购买和出售游戏道具、查看排行榜等。

本课程设计围绕“网络游戏推荐系统（可扩展为游戏装备交易等）”这一题目，设计并实现了一个基于 C 语言的控制台应用程序。系统面向普通玩家和管理员两类角色，支持用户注册、登录、信息管理、游戏推荐、好友管理、好友邀约、道具交易、交易记录查询以及排行榜等功能。系统能够根据用户的年龄、身份或职业、兴趣爱好、游戏时长、胜率等信息进行分类推荐，并依据游戏时长和胜率完成自动升级，从而体现网络游戏推荐与管理系统的综合功能。

系统采用模块化设计思想，以二进制文件作为主要数据存储方式，在保证实现难度适中的基础上，尽量满足题目中“菜单驱动、文件读写、复杂数据结构、不同角色权限”等要求。

### 2.系统运行环境

#### （1）硬件环境

- 普通个人计算机即可运行；
- 处理器：Intel i5 或同等级别处理器；
- 内存：4GB 及以上；
- 存储空间：能够保存程序文件与数据文件即可。

#### （2）软件环境

- 操作系统：Windows 10 / Windows 11；
- 编译器：GCC；
- 开发工具：Code::Blocks、VS Code 等；
- 程序类型：C 语言控制台程序。

### 3.功能需求分析

根据题目要求以及实际设计思路，本系统的功能需求主要分为以下几个部分。

#### （1）用户注册功能

系统应支持新用户注册。用户在注册时需要输入用户名、密码、年龄、身份或职业、兴趣爱好等基本信息。系统需要检查用户名是否重复，保证每个用户账号的唯一性，并为新注册用户分配唯一编号。

#### （2）用户登录功能

系统应支持用户通过账号和密码进行登录。为了更贴近题目要求，本系统在密码校验基础上增加了验证码输入环节。用户只有在用户名、密码和验证码均正确的情况下，才能登录系统并进入菜单界面。

#### （3）用户信息管理功能

系统应支持用户登录后查看并修改个人信息。可修改的内容包括密码、年龄、身份或职业、爱好、游戏时长、对局数、胜场数以及金币等。管理员则拥有更高权限，可以查看全部用户，并进行升级、降级和删除操作。

#### （4）用户画像功能

题目中提到推荐应基于用户画像，因此本系统将以下内容作为用户画像信息：

- 年龄；
- 身份或职业；
- 爱好；
- 游戏时长；
- 对局数；
- 胜场数；
- 胜率。

这些信息不仅用于个人信息展示，也会用于后续的推荐和自动升级逻辑。

#### （5）游戏推荐功能

系统应支持根据用户画像进行推荐。为了体现推荐逻辑，本系统在推荐时主要参考以下因素：

- 用户年龄是否满足游戏建议年龄；
- 用户爱好与游戏类型是否匹配；
- 用户身份或职业与游戏类型之间的简单对应关系；
- 用户游戏时长是否达到一定水平。

系统内置了若干热门网络游戏数据，如《王者荣耀》《原神》《第五人格》《和平精英》《蛋仔派对》《阴阳师》等，用于推荐演示。

#### （6）自动升级功能

根据题目要求，系统应支持依据用户的游戏娱乐时间或胜率进行升级。因此，本系统设计了自动升级规则：普通用户在满足“游戏时长达到一定标准”或“胜率达到一定标准”时，可以自动升级为 VIP 用户。该功能使系统更贴近题目要求中的“通过用户的游戏娱乐时间或胜率进行升级”。

#### （7）好友管理功能

系统应支持好友管理，包括：

- 添加好友；
- 删除好友；
- 查看好友列表；
- 发起好友邀约；
- 查看邀约记录。

好友邀约功能的加入，使系统在“交友”和“游戏娱乐相关任务”方面更加完整。

#### （8）交易功能

系统应支持与游戏资产相关的简单交易管理，主要包括：

- 查看市场中当前可交易的道具；
- 发布道具到市场；
- 购买他人发布的道具；
- 查看与自己相关的交易记录。

系统在交易成功后会自动修改买卖双方金币，并将交易信息写入交易记录文件。

#### （9）其他功能

为增强系统完整性，本系统还实现了金币排行榜功能，通过对用户金币进行排序，显示系统内排名靠前的用户。该功能与题目中“其他相关功能，例如游戏排行榜”相对应。

### 4.数据存储需求分析

系统采用二进制文件保存各类数据，主要包括：

- `users.dat`：保存用户信息；
- `games.dat`：保存游戏信息；
- `market.dat`：保存市场道具信息；
- `trade_log.dat`：保存交易记录；
- `invite_log.dat`：保存好友邀约记录；
- `id.dat`：保存编号信息；
- `seed.dat`：保存验证码伪随机种子。

上述文件共同构成了系统的数据存储基础，保证程序在多次运行之间能够保留用户、市场和记录信息，实现数据持久化。

## 二、总体设计

### 1.总体开发思想

本系统采用模块化和结构化设计思路，将程序划分为四个主要模块：crud 模块、manage 模块、trade 模块和 main 模块。这样的划分方式有利于将不同功能进行隔离，降低函数之间的耦合程度，也便于后期的调试和维护。

- **crud 模块**：负责底层数据文件的初始化、读取、保存和编号管理；
- **manage 模块**：负责用户管理、登录验证、个人信息处理、自动升级、好友与推荐功能；
- **trade 模块**：负责市场道具管理、购买交易以及交易记录；
- **main 模块**：负责系统入口、注册登录分流、主菜单和排行榜等功能。

系统整体执行流程如下：

1. 程序启动并初始化数据文件；
2. 若用户文件为空，则自动创建默认管理员账号；
3. 用户输入用户名；
4. 若账号存在，则进入登录流程；若账号不存在，则进入注册流程；
5. 登录成功后进入主菜单；
6. 用户根据菜单进入不同功能模块；
7. 用户执行相应操作后返回主菜单；
8. 用户选择退出，程序结束运行。

### 3.模块设计说明

#### （1）crud 模块设计说明

crud 模块是本系统的数据基础模块。由于系统采用二进制文件保存数据，因此几乎所有功能在执行前都需要先从文件读取数据，在执行后再写回文件。crud 模块主要实现以下内容：

- 初始化系统所需的各类数据文件；
- 生成用户和道具需要的唯一编号；
- 从文件读取用户、游戏、市场、交易记录和邀约记录；
- 将修改后的数组重新保存到文件中；
- 按用户名、用户编号和道具编号查找目标数据。

#### （2）manage 模块设计说明

manage 模块主要实现与“用户”有关的功能，包括：

- 用户注册与登录；
- 验证码生成与校验；
- 个人信息查看与修改；
- 自动升级逻辑；
- 好友管理与好友邀约；
- 基于画像的游戏推荐；
- 管理员对用户的管理。

这一模块是系统中最重要的业务模块之一。

#### （3）trade 模块设计说明

trade 模块主要实现游戏道具市场和交易功能。用户既可以在市场中查看和购买道具，也可以把自己的道具发布到市场中。交易完成后，系统自动记录交易日志，并完成金币变动。该模块体现了题目中“游戏装备交易等扩展内容”的要求。

#### （4）main 模块设计说明

main 模块负责程序运行的整体控制，包括程序启动、注册登录、菜单跳转、排行榜显示和退出等。该模块相当于整个系统的总入口，用于协调其他三个模块的调用顺序。

## 三、数据结构设计

### 1.用户结构体设计

本系统中最核心的数据结构是用户结构体：

```c
typedef struct {
    int id;
    char username[50];
    char passwd[50];
    int age;
    char identity[30];
    char hobby[30];
    int role;
    int coins;
    int game_hours;
    int play_count;
    int win_count;
    int friend_count;
    int asset_count;
    int friends[MAX_FRIENDS];
    int assets[MAX_ASSETS];
} users;
```

该结构体的作用较为全面，不仅保存了基本账号信息，还保存了画像信息、好友信息和资产信息。具体说明如下：

- `id`：用户编号，用于区分不同用户；
- `username`：用户名；
- `passwd`：密码；
- `age`：年龄；
- `identity`：身份或职业；
- `hobby`：兴趣爱好；
- `role`：角色信息，0 表示普通用户，1 表示 VIP，2 表示管理员；
- `coins`：金币数量；
- `game_hours`：游戏时长；
- `play_count`：总对局数；
- `win_count`：胜场数；
- `friend_count`：当前好友数量；
- `asset_count`：当前资产数量；
- `friends[]`：好友编号数组；
- `assets[]`：道具编号数组。

### 2.游戏结构体设计

```c
typedef struct {
    int game_id;
    char game_name[50];
    char game_type[30];
    char game_desc[100];
    int need_age;
} games;
```

该结构体用于保存游戏的基础资料，为推荐系统提供数据支持。字段主要包括游戏编号、游戏名称、游戏类型、游戏描述以及建议年龄。

### 3.市场结构体设计

```c
typedef struct {
    int item_id;
    char item_name[50];
    char item_type[30];
    int item_price;
    int seller_id;
} market;
```

市场结构体主要用于保存正在出售的道具信息。用户购买道具时，会根据该结构体的数据判断对应道具的价格和卖家。

### 4.交易记录结构体设计

```c
typedef struct {
    int buyer_id;
    int seller_id;
    char buyer_name[50];
    char seller_name[50];
    int item_id;
    char item_name[50];
    char item_type[30];
    int item_price;
} trade_log;
```

该结构体用于保存每一笔交易记录，方便用户查看自己的交易历史，也便于后续扩展统计功能。

### 5.好友邀约结构体设计

```c
typedef struct {
    int send_id;
    int recv_id;
    char send_name[50];
    char recv_name[50];
    char game_name[50];
} invite_log;
```

该结构体用于记录好友邀约关系。该部分虽然结构简单，但能够体现系统在“游戏娱乐相关任务”方面的扩展性。

### 6.数据组织方式分析

本系统采用“结构体数组 + 二进制文件”的方式组织数据，而没有使用链表等动态结构。这样设计主要基于以下几点考虑：

1. 更符合当前课程设计的知识范围；
2. 结构体数组更直观，便于理解和调试；
3. 二进制文件读写效率较高，能够较好地满足本系统需要；
4. 通过好友数组、资产数组等内容，也体现了一定的复杂数据结构使用。

## 五、功能实现

### 1.主要函数说明

#### （1）crud 模块主要函数

**1）void crud_init(void)**  
功能：初始化系统运行所需的数据文件。如果用户文件、市场文件、游戏文件、交易记录文件、邀约记录文件、编号文件和验证码种子文件不存在，则自动创建。

**2）void crud_write_default_games(void)**  
功能：向游戏文件中写入默认的热门网络游戏数据，用于系统推荐功能演示。

**3）int crud_next_id(void)**  
功能：读取当前编号并生成新的编号，同时更新编号文件。

**4）int crud_load_users(users list[])**  
功能：读取全部用户数据到数组中，并返回用户数量。

**5）void crud_save_users(users list[], int num)**  
功能：将用户数组整体写回用户数据文件。

**6）int crud_load_games(games list[])**  
功能：读取全部游戏数据。

**7）int crud_load_market(market list[])**  
功能：读取市场中的全部道具数据。

**8）void crud_save_market(market list[], int num)**  
功能：保存市场中的道具数据。

**9）void crud_add_trade_log(trade_log* log_data)**  
功能：追加写入一条交易记录。

**10）void crud_add_invite_log(invite_log* log_data)**  
功能：追加写入一条好友邀约记录。

#### （2）manage 模块主要函数

**1）int manage_user_register(users* user)**  
功能：完成用户注册，并初始化用户的基础属性。

**2）int manage_user_login(char username[], char passwd[])**  
功能：完成用户登录，包括密码校验和验证码校验。

**3）int manage_make_captcha(char username[], int id)**  
功能：生成一个四位数伪随机验证码，并更新验证码种子文件。

**4）int manage_get_win_rate(users* user)**  
功能：根据对局数和胜场数计算用户胜率。

**5）void manage_user_auto_upgrade(users* user)**  
功能：根据游戏时长和胜率完成普通用户到 VIP 用户的自动升级。

**6）void manage_user_show_info(char username[])**  
功能：显示用户的完整信息。
