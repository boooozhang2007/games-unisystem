## 一、需求分析

### 1.系统概述

随着网络游戏的发展，网络游戏已经成为很多人日常生活中常见的娱乐方式。现在市面上有各种各样的游戏，有竞技类、冒险类、休闲类和养成类等，适合不同年龄段的用户。在玩游戏的过程中，玩家除了娱乐之外，还会有社交需求和交易需求，比如添加好友、发起邀约、购买和出售游戏道具、查看排行榜等。

本课程设计围绕"网络游戏推荐系统"这一题目，设计并实现了一个基于 C 语言的控制台应用程序。系统面向普通玩家和管理员两类角色，支持用户注册、登录、信息管理、游戏推荐、好友管理、好友邀约、道具交易、交易记录查询以及排行榜等功能。系统可以根据用户的年龄、身份、爱好、游戏时长、胜率等信息来推荐游戏，还能根据游戏时长和胜率自动升级。

数据存储方面采用二进制文件，程序通过菜单驱动的方式运行，基本满足了题目中"菜单驱动、文件读写、复杂数据结构、不同角色权限"等要求。

### 2.系统运行环境

#### （1）硬件环境

- 普通个人计算机即可运行；
- 处理器：Intel i5 或同等级别处理器；
- 内存：4GB 及以上；
- 存储空间：能够保存程序文件与数据文件即可。

#### （2）软件环境

- 操作系统：Windows 10 / Windows 11；
- 编译器：GCC；
- 开发工具：VS Code；
- 程序类型：C 语言控制台程序。

### 3.功能需求分析

根据题目要求和实际设计思路，本系统的功能需求主要分为以下几个部分。

#### （1）用户注册功能

系统支持新用户注册。用户在注册时需要输入用户名、密码、年龄、身份或职业、兴趣爱好等基本信息。系统会检查用户名是否重复，保证每个用户的唯一性，并为新用户分配唯一编号。

#### （2）用户登录功能

系统支持用户通过账号和密码进行登录。在密码校验的基础上还增加了验证码输入环节，用户需要在用户名、密码和验证码都正确的情况下才能登录成功。

#### （3）用户信息管理功能

用户登录后可以查看并修改个人信息，包括密码、年龄、身份或职业、爱好、游戏时长、对局数、胜场数以及金币等。管理员拥有更高权限，可以查看全部用户，并进行升级、降级和删除操作。

#### （4）用户画像功能

题目中提到推荐应基于用户画像，因此本系统将以下内容作为用户画像信息：

- 年龄；
- 身份或职业；
- 爱好；
- 游戏时长；
- 对局数；
- 胜场数；
- 胜率。

这些信息用于个人信息展示，也会用在推荐和自动升级的逻辑里。

#### （5）游戏推荐功能

系统可以根据用户画像进行游戏推荐，推荐时主要考虑以下因素：

- 用户年龄是否满足游戏建议年龄；
- 用户爱好与游戏类型是否匹配；
- 用户身份或职业与游戏类型的对应关系，比如学生推荐竞技类游戏；
- 用户游戏时长是否达到一定水平。

系统内置了6款热门网络游戏数据，包括《王者荣耀》《原神》《第五人格》《和平精英》《蛋仔派对》《阴阳师》，用于推荐演示。

#### （6）自动升级功能

根据题目要求，系统支持依据用户的游戏时长或胜率进行升级。具体规则是：普通用户的游戏时长达到 100 小时，或者胜率达到 60% 时，可以自动升级为 VIP 用户。

#### （7）好友管理功能

系统支持好友管理功能，包括：

- 添加好友；
- 删除好友；
- 查看好友列表；
- 发起好友邀约；
- 查看邀约记录。

好友邀约功能的加入，使系统在"交友"和"游戏娱乐相关任务"方面更加完整。

#### （8）交易功能

系统支持游戏资产相关的交易管理，主要包括：

- 查看市场中当前可交易的道具；
- 发布道具到市场；
- 购买他人发布的道具；
- 查看与自己相关的交易记录。

交易成功后系统会自动修改买卖双方的金币，并将交易信息写入交易记录文件。

#### （9）其他功能

系统还实现了金币排行榜功能，通过对用户金币进行排序，显示排名靠前的用户。对应题目中"其他相关功能，例如游戏排行榜"的要求。

### 4.数据存储需求分析

系统采用二进制文件保存各类数据，主要包括：

- `users.dat`：保存用户信息；
- `games.dat`：保存游戏信息；
- `market.dat`：保存市场道具信息；
- `trade_log.dat`：保存交易记录；
- `invite_log.dat`：保存好友邀约记录；
- `id.dat`：保存编号信息。

这些文件保证程序在多次运行之间能够保留用户、市场和记录信息，实现数据持久化。

## 二、总体设计

### 1.总体开发思想

本系统将程序划分为四个部分：crud 部分、manage 部分、trade 部分和 main 部分。这样分开来写比较清楚，找问题也方便。

- **crud 部分**：负责底层数据文件的初始化、读取、保存和编号管理；
- **manage 部分**：负责用户管理、登录验证、个人信息处理、自动升级、好友管理与推荐功能；
- **trade 部分**：负责市场道具管理、购买交易以及交易记录；
- **main 部分**：负责系统入口、注册登录分流、主菜单和排行榜等功能。

系统整体执行流程如下：

1. 程序启动并初始化数据文件；
2. 若用户文件为空，则自动创建默认管理员账号；
3. 用户输入用户名；
4. 若账号存在，则进入登录流程；若账号不存在，则进入注册流程；
5. 登录成功后进入主菜单；
6. 用户根据菜单进入不同功能；
7. 操作完成后返回主菜单；
8. 用户选择退出，程序结束运行。

### 3.模块设计说明

#### （1）crud 部分设计说明

crud 部分是系统的数据基础部分。由于系统采用二进制文件保存数据，基本上所有功能在执行前都需要先从文件读取数据，执行后再写回文件。crud 部分主要实现以下内容：

- 初始化系统所需的各类数据文件；
- 生成用户和道具需要的唯一编号；
- 从文件读取用户、游戏、市场、交易记录和邀约记录；
- 将修改后的数组重新保存到文件中；
- 按用户名、用户编号和道具编号查找目标数据。

#### （2）manage 部分设计说明

manage 部分主要实现与用户有关的功能，包括：

- 用户注册与登录；
- 验证码校验；
- 个人信息查看与修改；
- 自动升级逻辑；
- 好友管理与好友邀约；
- 基于画像的游戏推荐；
- 管理员对用户的管理。

这是系统中功能最多的一个部分。

#### （3）trade 部分设计说明

trade 部分主要实现游戏道具市场和交易功能。用户可以在市场中查看和购买道具，也可以把自己的道具发布到市场中。交易完成后，系统自动记录交易日志，并完成金币变动。

#### （4）main 部分设计说明

main 部分负责程序运行的整体控制，包括程序启动、注册登录、菜单跳转、排行榜显示和退出等。其他三个部分的函数都是从这里调用的。

## 三、数据结构设计

### 1.用户结构体设计

本系统中最核心的数据结构是用户结构体：

```c
typedef struct {
    int id;
    char username[50];
    char passwd[50];
    int age;
    char identity[30];
    char hobby[30];
    int role;
    int coins;
    int game_hours;
    int play_count;
    int win_count;
    int friend_count;
    int asset_count;
    int friends[MAX_FRIENDS];
    int assets[MAX_ASSETS];
} users;
```

该结构体保存了用户的基本账号信息、画像信息、好友信息和资产信息。具体说明如下：

- `id`：用户编号，用于区分不同用户；
- `username`：用户名；
- `passwd`：密码；
- `age`：年龄；
- `identity`：身份或职业；
- `hobby`：兴趣爱好；
- `role`：角色信息，0 表示普通用户，1 表示 VIP，2 表示管理员；
- `coins`：金币数量；
- `game_hours`：游戏时长；
- `play_count`：总对局数；
- `win_count`：胜场数；
- `friend_count`：当前好友数量；
- `asset_count`：当前资产数量；
- `friends[]`：好友编号数组；
- `assets[]`：道具编号数组。

### 2.游戏结构体设计

```c
typedef struct {
    int game_id;
    char game_name[50];
    char game_type[30];
    char game_desc[100];
    int need_age;
} games;
```

该结构体用于保存游戏的基础资料，字段包括游戏编号、游戏名称、游戏类型、游戏描述以及建议年龄。推荐游戏的时候会用到这些数据。

### 3.市场结构体设计

```c
typedef struct {
    int item_id;
    char item_name[50];
    char item_type[30];
    int item_price;
    int seller_id;
} market;
```

市场结构体用于保存正在出售的道具信息。用户购买道具时，会根据该结构体判断道具的价格和卖家。

### 4.交易记录结构体设计

```c
typedef struct {
    int buyer_id;
    int seller_id;
    char buyer_name[50];
    char seller_name[50];
    int item_id;
    char item_name[50];
    char item_type[30];
    int item_price;
} trade_log;
```

该结构体用于保存每一笔交易记录，方便用户查看自己的交易历史。

### 5.好友邀约结构体设计

```c
typedef struct {
    int send_id;
    int recv_id;
    char send_name[50];
    char recv_name[50];
    char game_name[50];
} invite_log;
```

该结构体用于记录好友邀约，包括谁邀请谁、玩什么游戏。

### 6.数据组织方式分析

本系统采用"结构体数组 + 二进制文件"的方式组织数据，没有使用链表。主要原因是：

1. 结构体数组比较简单直观，写起来方便，也容易调试；
2. 二进制文件读写效率比较高，能满足系统的需要；
3. 好友数组、资产数组等内容也体现了一定的复杂数据结构使用。

## 五、功能实现

### 1.主要函数说明

#### （1）crud 部分主要函数

**1）void crud_init(void)**  
功能：初始化系统运行所需的数据文件。如果用户文件、市场文件、游戏文件、交易记录文件、邀约记录文件和编号文件不存在，则自动创建。

**2）void crud_write_default_games(void)**  
功能：向游戏文件中写入默认的6款热门网络游戏数据，用于推荐功能演示。

**3）int crud_next_id(void)**  
功能：读取当前编号并加1生成新的编号，同时更新编号文件。

**4）int crud_load_users(users list[])**  
功能：从文件中读取全部用户数据到数组中，并返回用户数量。

**5）void crud_save_users(users list[], int num)**  
功能：将用户数组整体写回用户数据文件。

**6）int crud_load_games(games list[])**  
功能：读取全部游戏数据。

**7）int crud_load_market(market list[])**  
功能：读取市场中的全部道具数据。

**8）void crud_save_market(market list[], int num)**  
功能：保存市场中的道具数据。

**9）void crud_add_trade_log(trade_log* log_data)**  
功能：追加写入一条交易记录。

**10）void crud_add_invite_log(invite_log* log_data)**  
功能：追加写入一条好友邀约记录。

#### （2）manage 部分主要函数

**1）int manage_user_register(users* user)**  
功能：完成用户注册，设置好默认属性后保存到文件。

**2）int manage_user_login(char username[], char passwd[])**  
功能：完成用户登录，包括密码校验和验证码校验。

**3）int manage_get_win_rate(users* user)**  
功能：根据对局数和胜场数计算用户胜率，返回整数百分比。

**4）void manage_user_auto_upgrade(users* user)**  
功能：检查用户的游戏时长和胜率，满足条件则从普通用户升级为 VIP。

**5）void manage_user_show_info(char username[])**  
功能：显示用户的完整个人信息。

**6）int manage_user_edit_info(char username[])**  
功能：修改用户的密码、年龄、身份、爱好、游戏时长、胜负记录和金币。

**7）int manage_friend_add(char username[])**  
功能：添加好友，双方都会加上对方。

**8）int manage_friend_remove(char username[])**  
功能：删除好友，双方都会删掉对方。

**9）int manage_friend_invite(char username[])**  
功能：向好友发起游戏邀约，并保存邀约记录。

**10）void manage_friend_show_invite(char username[])**  
功能：显示与当前用户有关的邀约记录。

**11）void manage_show_recommend_games(char username[])**  
功能：根据用户画像信息推荐适合的游戏。

#### （3）trade 部分主要函数

**1）void trade_show_market(void)**  
功能：显示当前市场中的全部道具信息。

**2）int trade_publish_item(char username[])**  
功能：上架道具到市场。

**3）int trade_buy_item(char username[])**  
功能：购买市场中的道具，完成金币更新与交易记录保存。

**4）void trade_show_log(char username[])**  
功能：显示与当前用户相关的交易记录。

#### （4）main 部分主要函数

**1）int main_show_menu(int role)**  
功能：显示主菜单，管理员和普通用户看到的菜单不同。

**2）void main_show_ranking(void)**  
功能：根据金币数量对用户进行排序，显示排行榜。

**3）void main_first_register(char username[])**
